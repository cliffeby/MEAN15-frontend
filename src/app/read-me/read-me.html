Loan & Offers Management System - Project Overview

Matches Feature

Overview

The Matches feature enables users to create, view, edit, and manage golf matches, including player lineups, scorecards, and match status. It is tightly integrated with NgRx for state management in the frontend.

Frontend (Angular)

  • Location: Components: src/app/components/match-list/, match-form/, match-edit/, match-selection-dialog/
  • Service: src/app/services/matchService.ts
  • Model: src/app/models/match.ts
  • Store: src/app/store/match/ (actions, reducers, selectors, effects)
  • Routes: /matches, /matches/edit/:id, /matches/new
  • List, add, edit, delete, and view matches
  • Player lineups, scorecard assignment, match results
  • Match selection dialog for other features (e.g., score entry)
  • Access via sidebar: Click "Matches" to view the list
  • Data is managed via NgRx store and fetched from the backend via MatchService

NgRx Store Usage

  • Actions: Define match-related events (load, create, update, delete)
  • Reducers: Manage match state (list, selected match, loading, error)
  • Selectors: Query match state for components (get all matches, get selected match)
  • Effects: Handle side effects (API calls for matches)
  • Benefits:
    • Centralized state management for matches
    • Automatic UI updates when match data changes
    • Simplified data flow and debugging
    • Decouples API logic from UI components

Backend (Node.js/Express/Mongoose)

  • Location: Model: backend/models/Match.js, Controller: backend/controllers/matchController.js, Routes: backend/routes/matchRoutes.js
  • API Endpoints:
    • GET /api/matches — List all matches
    • GET /api/matches/:id — Get match by ID
    • POST /api/matches — Add new match
    • PUT /api/matches/:id — Update match
    • DELETE /api/matches/:id — Delete match
  • Data Model: Fields: name, date, scorecardId, lineUps, status, results, etc.
  • Validation and business logic handled in controller/model

How to Use

  • Use the sidebar to navigate to "Matches"
  • All match management actions are available from the Matches page
  • Changes are reflected in both frontend and backend

Additional Notes

  • Match data is used in score entry, leaderboards, and other features
  • NgRx ensures consistent state and UI updates
  • For technical details, see the respective service, model, controller, and store files

For further help, contact the project maintainer or see the full API documentation.

Members Feature

Overview

The Members feature allows users to view, add, edit, and manage members in the application. It is available in both the frontend (Angular) and backend (Node.js/Express/Mongoose).

Frontend (Angular)

  • Location: Components: src/app/components/member-list/, member-form/, member-edit/
  • Service: src/app/services/memberService.ts
  • Model: src/app/models/member.ts
  • Routes: /members, /members/edit/:id, /members/new
  • List, add, edit, delete, and search members
  • Member selection dialog for other features (e.g., match setup)
  • Access via sidebar: Click "Members" to view the list
  • Data is fetched from the backend via MemberService

Backend (Node.js/Express/Mongoose)

  • Location: Model: backend/models/Member.js, Controller: backend/controllers/memberController.js, Routes: backend/routes/memberRoutes.js
  • API Endpoints:
    • GET /api/members — List all members
    • GET /api/members/:id — Get member by ID
    • POST /api/members — Add new member
    • PUT /api/members/:id — Update member
    • DELETE /api/members/:id — Delete member
  • Data Model: Fields: name, email, league, handicap, status, etc.
  • Validation and business logic handled in controller/model

How to Use

  • Changes are reflected in both frontend and backend

Additional Notes

Project Overview

This system is a full-featured platform for administrators and customers, enabling management of loans, offers, users, and dashboards. It provides secure authentication, dynamic dashboards, and CRUD operations for loans and offers.

Tech Stack & Backend Setup

  • Frontend:Angular 20
  • Backend: Node.js with Express.js
  • Database: MongoDB using Mongoose ODM
  • Authentication: JWT token-based authentication
  • Password Security: bcrypt for password hashing
  • API Endpoints: RESTful endpoints for loans, offers, and authentication
  • Middleware: Helmet for security headers, CORS, rate limiting, compression, and error handling
  • Logging: Morgan for request logging (development and production modes)
  • Deployment Ready: Modular route and controller structure with production-ready error handling

Key Features (End-User Readable)

Dashboard Overview

  • Real-time total loans, total offers, and total customers.
  • Quick overview of loan statuses (Pending, Approved, Declined) via charts.
  • Direct access to loan and offer lists from the dashboard.

Golf Management (CRUD)

  • Create Golf Entry: Admins can add new golf entries with amount, term, interest rate, and status.
  • Read/List Golf Entries: View all golf entries in searchable, sortable, paginated tables.
  • Update Golf Entry: Edit golf entry details or status seamlessly.
  • Delete Golf Entry: Remove golf entries with confirmation prompts.
  • Golf Entry Detail Page: View all golf entry information including customer details.

Offer Management (CRUD with NgRx)

  • Create Offer: Admins can add new offers using a reactive form integrated with NgRx.
  • Read/List Offers: Offers displayed in elegant tables or cards with real-time updates.
  • Update Offer: Edit existing offers; state management handled by NgRx store and effects.
  • Delete Offer: Remove offers with confirmation; NgRx updates state automatically.
  • State Management: Offers CRUD fully integrated with NgRx actions, reducers, selectors, and effects.

Authentication & User Management

  • Secure login/logout for admins and customers.
  • Role-based access: Admins manage all loans and offers; customers view their own data.
  • JWT token-based authentication for API calls.
  • Password hashing with bcrypt for secure storage.

Form Handling & Validation

  • Reactive forms for offers and loans with real-time validation and feedback.
  • Error messages and notifications for invalid input or failed actions.

Interactive Tables & UI

  • Sorting, searching, and pagination for loans and offers.
  • Material tables and cards with responsive layouts.
  • Dynamic filtering by multiple fields (status, amount, customer, offer type).

Notifications & Feedback

  • Success and error notifications on loan and offer actions.
  • Confirmation prompts for critical actions like deletion.

User-Friendly Design

  • Clean, responsive UI using Angular Material components.
  • Toolbar with navigation and action buttons for better workflow.
  • Vertical and card-based forms for better readability.

Additional Notes

  • All components are responsive and enterprise-ready.
  • Supports scaling with additional modules like Reports, Analytics, and Notifications.
  • Seamless integration between CRUD operations and state management via NgRx.
  • Backend is production-ready with secure authentication, proper error handling, and modular design.
